Sukhumwit Crawler Spider Settings

These options control how the spider operates. These are generally advanced settings that should rarely need to be modified.

Note: Changes to any of the fields on this screen do not take effect until the Apply option pressed.

Also, there is currently no way to save these options between runs.

  • Workload Manager: The spider can operate storing the lists of visited pages and pages remaining to visit either in memory or in a data base. Use this field to specify either the Memory Based or Database workload manager for the spider.
  • Web Connections:
    • Timeout: How many milliseconds to wait when downloading pages. Defaults to 90000 (90 seconds).
    • User Agent: What user agent should be reported to the web site. This generally allows the web site to determine what browser is in use. Leave blank (default) for no reporting of a user agent.
    • Max URLs: This field limits the number of URLs that will be added to the workload during processing. Once the crawler reaches this maximum it refuses to add any more URLs that are found to its workload. This is useful for an intial test run on a site. Leave blank or specify -1 for no limit.
  • Background Tasks: Pages that are visited by the spider are processed in the background. See Background Operation for more details.
    • Core Pool Size: The core thread pool size. This indicates the number of active parses that are allocated to run in the background.
    • Max Pool Size: The maximum thread pool size.
    • Keep Alive Time: How long, in seconds, to keep inactive threads alive. Defaults to 60 (seconds).
  • Database Settings: These fields are only enabled when the database workload manager is selected. These options tell the spider where the database is located and how to connect to it:
    • Database URL: Advanced setting that specifies the location and name of of workload database as it is known to the operating system.
    • Database Driver: Advanced setting that specifies the code that will mediate between the database and the application. Defaults to the Sun Microsystems JDBC driver (sun.jdbc.odbc.JdbcOdbcDriver).
    • Database User ID: The user id (name) to access the database.
    • Database Password: The password to access the database.
  • Resume or Restart: Only available with the database workload manager,
    • Restart: specifies that when the spider starts up it should clear the workload and start over with the crawl.
    • Resume: specifies that the spider should resume processing its workload where it left of on the previous run
