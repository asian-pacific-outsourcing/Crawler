Sukhumwit Crawler Input Panel

Contents

 

This panel accepts the basic user inputs which tell the spider where to begin its search, and what to look for.

The following fields and buttons should be verified before you click the "Start" button, to begin the crawl.

  • Host: Select the web host configuration that you wish to search. This will cause several other fields to be filled in and will select the custom parser designed to look for contacts on that host type.
  • Root URL(s): Specify the first page where the crawler should begin its search.
    Note: If this field contains %d it will be treated as a pattern, allowing several similar URLs to be scheduled for processing at the beginning of the search. This feature combined with the exclusion feature described below can significantly improve the crawler's performance. For more information see Specifying Root URL Patterns below.
  • Range: Use these fields to specify an inclusive range of non-negative integers to be substituted into the pattern specified in the Root URL(s) field. For more information see Specifying Root URL Patterns below.
  • MaxDepth: The maximum depth to search pages. Enter -1 to indicate no maximum depth (the spider can run forever!). Defaults to 4.
  • Exclude URLs containing...: Use this field to specify an arbitrary number of strings, which if encountered by the crawler will be skipped. Enter one string per line and do not leave any blank lines. For example if you typed in the string copy on a line, any URLs, such as http://www.yahoo.com/copyright.html, which contain that string will not be searched for links or contacts. If you click the start button, these values will be saved to disk. They are restored the next time you select this host.

Specifying Root URL Patterns

The Root URL(s) field is treated as a pattern whenever %d is present in the field. This causes the spider to imediately schedule multiple URLs for scanning upon starting.

For example if you specify the Root URL as http://www.yahoo.com/fred%d.html and enter the range as 3 - 5 the following URLs would be scheduled when the spider is launched:

  • http://www.yahoo.com/fred3.html
  • http://www.yahoo.com/fred4.html
  • http://www.yahoo.com/fred5.html
If you combine this with excluding the string fred, in the Exclude URLs containing... field you ensure that fred6.html, fred7.html, etc. are not searched on this run.

A Variation

If you want to specify leading zeros you can use %0xd, where the x indicates the width of the numbers. For example, specifying the range 8 - 10 with the pattern http://www.yahoo.com/fred%04d.html would yield the following URLs:

  • http://www.yahoo.com/fred0008.html
  • http://www.yahoo.com/fred0009.html
  • http://www.yahoo.com/fred0010.html
